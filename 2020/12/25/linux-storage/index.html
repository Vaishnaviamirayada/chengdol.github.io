首先需要理解加入一个新的disk到系统之后，发生了什么，以及需要做什么工作才能使用这个新加入的disk。可以参考一下这篇文章. 主要用到了fdisk(partition), mkfs(make filesystem), mount or fstab(make accessable)

新加入的disk 的device file 如何命名的, Name conventions of device file, see here, 比如/dev/sda, /dev/sdb, etc.

总的来看，可以用这样的顺序去观察storage system:

  • lsblk, blkid 查看block device的大致状态，fs, disk, part, mount point等
  • /etc/fstab 查看是否persistent 以及 mount option 是否合适
  • mount 查看一下defaults mount option 具体内容是什么
  • df -k/-i 查看mount point space使用情况
  • dd 测试I/O performance (或者结合iperf3如果是NFS 之类的分布式存储)

lsblk and blkid are used to identify block storages (linux also has character device).

Formating

Formatting partition or disk is to make a filesystem on it.

[ ] Ordinary file can also be formatted and mounted.

The persistent mounts are in /etc/fstab file on boot, If you type mount command on shell, you will see several mount points, like above proc, sysfs, debugfs. These are not in /etc/fstab, systemd mounts them on boot automatically.

For the mount options specific to file system type, see man fstab and mount.

You can unmount by umount command, filesystem cannot be unmounted while in use: files are open, process has dir in it. Can check by lsof command.

Filesystem Types

Commonly used ones: ext2/3/4, xfs, Btrfs. they have different properties.

Note that sometime you cannot create file because of no space left on device, but when you check df -k ., it is not full, check df -i . may help, you may use up the inode capacity even if there are lots of space still.
