Groovy Gradle and Jenkins

Posted on 2020-05-05 | Edited on 2020-05-11 | In InPorgress

This blog is still in progress…

最近在做一些pipeline的工作，需要快速上手，主要涉及到的:
Jenkins
gradle
Groovy
现在不清楚的地方是各自文件的语法。
jenkins how to start from scratch, how to configure?
先从groovy，gradle入手，然后jenkins。

Groovy

Books, course and presentations, recommend:
https://groovy-lang.org/learn.html#books

http://groovy-lang.org/syntax.html
alternative java language

DSL: domain specific language
sdkman very good installation toolkit (also for gradle!)

others:

Gradle

don’t see latest book, please refer to official doc
https://gradle.org/guides/
Ant and Maven are also build tools, gradle wrapper, incremental builds.

  1. build by convention
  2. Groovy DSL
  3. supports dependencies
  4. support mutli-project build
  5. easily customizable

To install Gradle, using sdkman tool.

Jenkins

Other interesting project: Tekton, Jenkinx (k8s involved)
Declarative Pipelines: https://www.jenkins.io/doc/book/pipeline/syntax/

Install jenkins, must have right openjdk installed.

or using rpm download for centos/redhat, then using systemctl to start jenkins
https://www.jenkins.io/doc/book/installing/#red-hat-centos

then you can open the web version by public ip:8080.

Jenkins use file system to store everything, if using systemd, the configuration is in /var/lib/jenkins. You can backup this folder, or if you want to wipe it out, then run systemctl restart jenkins, then jenkins is in init state.

Even don’t need to use Jenkins UI to create project, you can directly mkdir and files in the Jenkins directory, then go to Manage Jenkins -> Reload Configuration from Disk.

Demo Jenkins URL (from my fyre node):
http://9.30.166.179:8080/
there are some failures in installing plugins. Can fix the failure by Plugin Manager

Run jenkins in docker container

这块很有意思，加入容器后就太灵活了。
https://hub.docker.com/r/jenkins/jenkins
you can run several jenkins version in your machine, for testing new features, upgrading…
You may need to customize the jenkins docker image.

Creating app build

freestyle project -> pipeline (series of freestyle project)

Jenkins Workspace, you can see it in console output or click Workspace icon in your project dashboard. Everything from git clone to build result is put in your project workspace. Every build will overrode the previous one. (you can use tar command to backup and restore this workspace)

To store the build artifact, use Post-build Action in configure. for example, you want to archive some jar or zip files. then after build is done, these archives will show in the build page.

Build trend

right to Build History list, there is a trend button, click it will see the build time history statistics and distribution.

Testing and Continuous integration

Now start the pipeline job type. After creating a pipeline job, you will see pipeline syntax button in the page, it contains necessary resources to start. You can also use Copy from to create a pipeline from another, for quick start

一般来说，使用在git中的declarative pipeline script (for example: Jenkins file)去构造pipeline. 在pipeline configure中设置这一属性。

这里有关于declarative pipeline的介绍视频: https://www.jenkins.io/solutions/pipeline/

Add slave nodes

To add slaves, you can add New Node in Build Executor Status, you need to setup credentials and know_hosts file for ssh connection. (如何node没有被发现，会显示错误，根据错误指示排查问题即可)

Pipeline steps

如果没用declarative pipeline script， 而是通过pipeline generator去设置script (可以在这个script中定义各种变量)，则常用的Steps:

  • node: Allocate node
    Jenkins use master <-> agent model, you can configure tasks to be executed in agent node.

  • stage: Stage

  • git: Git
  • dir: Change Current Directory
  • sh: Shell Script
  • step: General Build Step
  • emailext: Extended Email

Triggering auto build

在pipeline configure中有Builder Triggers可以选择:

  • Build after other projects are built
  • Build periodically
  • Poll SCM
  • Disable this project
  • Quiet period
  • Trigger builds remotely

Email Notification

Using emailext: Extended Email，可以用groovy函数包装，传入email的subject以及内容再调用。

Managing plugins

Manage Jenkins -> Manage Plugins, then you can select and install plugin in Available section. For example:

  • Pipeline (如果这个没安装，则不会在UI中显示pipeline的动态流程图)

  • Html publisher (常用于发布unit test后的html report，这些html文件其实是被相关test生成好的)

  • Green Balls (show green color for success)

  • Blue Ocean (embedded site with new Jenkins UI)

Pipeline compatible plugins: https://github.com/jenkinsci/pipeline-plugin/blob/master/COMPATIBILITY.md

在初始化设置Jenkins的时候，有可能有plugins安装失败，可以自己在Manage plugin中安装，然后restart Jenkins (关于restart Jenkins请在没有job运行的情况下进行，不同的安装方式restart的方法不同，或者在安装plugin的时候选择restart jenkins after install), for example: systemctl restart jenkins, 这可以消除控制面板上的plugin failure警告。。
