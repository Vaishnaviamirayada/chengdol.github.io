Elastic Stack

https://www.elastic.co/elastic-stack
The Elastic Stack is one of the most effective ways to leverage open source technology to build a central logging, monitoring, and alerting system for servers and applications.

Elasticsearch: distributed, fast, highly scalable document database.
Logstash: Aggregates, filters and supplyments log data, forwards them to Elasticsearch.
Kibana: Web-based front-end to visualize data.
Beats: lightweight utilities for reading logs from a varity of sources, sends data to Logstash or other backends.
Altering: Send notifications to email, slack, pagerduty so on and so forth.

Elastic stack vs Prometheus:
ELK is general-purpose no-sql stack can be used for monitoring, aggregating all the logging and shipping to elastic search for ease of browsing all the logging and similar things.

Prometheus is dedicated monitoring system, alongside with service discovery consul and alert-manager, the right tool for the jobs.

Install Logstash

https://www.elastic.co/downloads/logstash

Java runtime is required.

Installing by yum repos, by default the app location is in /usr/share/logstash, enable and start by systemd. A simple test to verify the correction:

Using systemd to start Logstash will load the configuration specified in /etc/logstash/logstash.yml. Using command to start Logstash is not recommended, usually for testing.

In /etc/logstash/logstash.yml file, you can set the value of path.config field so Logstash know where to pick the pipeline configurations.
https://www.elastic.co/guide/en/logstash/current/logstash-settings-file.html

Beats

https://www.elastic.co/beats/
Beats can output data to Elasticsearch, Logstash and Redis. But usually we send data to Logstash (pre-processing) then forward to Elasticsearch.

Each Beat has configure yaml file with detailed configuration guideline. For example, in the configure yaml file, comment out Elasticsearch output, use Logstash output.

  • Filebeat: text log files
  • Metricbeat: OS and applications
  • Packetbeat: network monitoring
  • Winlogbeat: windows event log
  • Libbeat: write your own
