Docker Daemon Log

Posted on 2019-12-02 | Edited on 2019-12-03 | In Docker

When I was working on securing docker registry, I followed the instructions but when run docker push I always get x509: certificate signed by unknown authorit error, this means the self-signed certificate is still not identified by docker daemon.

This time to get more detail information, need to check the docker daemon log.

How to enable debug mode?

By default, the debug is mute, not much helpful info from logs, check here: https://docs.docker.com/config/daemon/, enable debugging section.

Edit the daemon.json file, which is usually located in /etc/docker/. You may need to create this file, if it does not yet exist.

Where is the log?

https://stackoverflow.com/questions/30969435/where-is-the-docker-daemon-log

I am in Red Hat, from /var/log/messages file I clearly see that the docker daemon pick certificate under /etc/docker/certs.d/<domain, no port number!> folder.

If your OS is using systemd, the journalctl command can help, but the output from container is also dumping here, see this issue: https://github.com/moby/moby/issues/23339.
